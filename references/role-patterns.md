# Role-Specific Question Patterns

High-frequency interview questions organized by role group and question category. Use these as the foundation when generating interview questions — adapt them to the candidate's actual resume details rather than using them verbatim.

---

## AI PM / Product Manager

### 数字类高频问题
- 你访谈了N名用户——样本怎么选的？能代表目标用户群吗？
- 命中率/转化率从X%提升到Y%，两轮迭代分别改了什么？每次提升了多少个百分点？
- P95 延迟是Xs，用户真的能接受吗？你在产品层如何处理等待体验？
- 回访率/留存率的分母是什么？统计窗口是多久？是注册用户还是活跃用户？
- 从立项到上线只有N天，功能完整度怎么样？主动砍掉了什么？技术债在哪里？

### 决策类高频问题
- 为什么选这个技术方案/模型/工具，而不是更简单的替代方案？
- 为什么用 Golden Set / 离线评测而不是 A/B 测试？
- Pipeline 中哪一步是你最纠结要不要加的？如果只能保留一步你选哪个？
- 如果只能保留一个功能，你会保留哪个？为什么是它而不是其他的？
- 你的 MVP 范围是怎么划定的？哪些需求是你主动推后的，为什么？

### 方法论类高频问题
- 你的评测体系具体是怎么设计的？如何防止在评测集上过拟合？
- PRD 和开发约束文档（如 CLAUDE.md）分别解决什么问题？为什么需要两个文档？
- 你是如何把用户访谈洞察转化为具体产品决策的？中间经历了几轮方案迭代？
- 你怎么定义"首位命中"或"采用"这个行为？判断标准是什么？谁来判？

## QA / 测试工程师

### 数字类高频问题
- 你的测试覆盖率是多少？是行覆盖、分支覆盖，还是业务路径覆盖？
- 自动化测试占比多少？剩余的手工测试为什么没有自动化？
- 线上 Bug 逃逸率是多少？逃逸的 Bug 主要集中在哪类场景？
- 你的 CI pipeline 执行时长是多久？超过这个时长开发会绕过它吗？
- 你在X周内完成了Y个模块的测试——是什么让你能做到这个速度？

### 决策类高频问题
- 你怎么决定一个场景该自动化还是手工测试？判断标准是什么？
- 在时间压力下，你如何决定测什么、不测什么？风险优先级怎么排？
- 你选择当前测试框架的依据是什么？当时考虑过哪些替代方案？
- 发现高风险 Bug 但临近上线，你会怎么决策是否阻塞发布？你有没有被迫妥协的经历？

### 方法论类高频问题
- 你的测试用例设计方法是什么？等价类划分、边界值分析、探索性测试如何结合？
- 你如何与开发协作让 Bug 在早期被发现，而不是堆积到测试阶段？
- 你如何衡量一个版本的质量是否达到上线标准？你用什么指标做上线决策？

## 销售 / 商务拓展

### 数字类高频问题
- 你上个季度配额完成率是多少？当季的市场环境和竞争态势如何？
- 你的平均销售周期是多久？比行业均值长还是短？原因是什么？
- 你管理了多少个商机？最终赢单率大概是多少？
- 你拿下的最大单是多少金额？这个单子跟了多久？转折点在哪里？
- 你的客户续约率/复购率是多少？哪些客户流失了？流失原因是什么？

### 决策类高频问题
- 你如何判断一个线索值不值得深入跟进？你的客户资格判断框架是什么？
- 面对竞品价格更低，你如何说服客户选你？你有没有赢过价格战的案例？
- 你如何处理内部资源不够支持某个大客户的情况？
- 签单前最后一个月客户突然冷掉，你会怎么处理？实际案例呢？

### 方法论类高频问题
- 你用什么方法管理销售漏斗和跟进节奏？用什么工具？
- 你如何快速建立对一个新行业/新客户类型的了解？
- 你如何在维护存量客户和开拓新客户之间分配时间和精力？

## 研究 / 算法工程师

### 数字类高频问题
- 你的模型在什么数据集上达到了什么指标？这个数据集有多大、怎么来的？
- 你的方案比 baseline 提升了多少？baseline 是怎么选的？为什么选它作为对比？
- 训练数据量是多少？其中有多少是标注数据？标注质量如何保证？
- 你的实验跑了多少组？最终选了哪个参数配置？依据是什么？有没有超参搜索策略？

### 决策类高频问题
- 为什么选这个模型架构而不是更简单或更复杂的替代方案？
- 你如何在模型精度和推理速度之间做取舍？取舍的边界在哪里？
- 实验结果持续不如预期时，你如何决定是继续调整还是放弃换方向？
- 你如何决定这项研究的核心 contribution 是什么？有没有调整过方向的经历？

### 方法论类高频问题
- 你如何设计实验来隔离地验证你的核心假设？
- 如何防止在测试集上过拟合或数据泄露？你的 hold-out 策略是什么？
- 你的研究成果如何走向工程落地？在这个过程中遇到过哪些挑战？

## 工程 / 后端开发

### 数字类高频问题
- 你负责的系统 SLA 是多少？最近一次故障是什么原因？怎么恢复的？
- 接口 P99 延迟是多少？你做过哪些优化？效果如何？
- 你的代码单测覆盖率是多少？这个覆盖率是主动选的，还是自然形成的？
- 这个功能的开发周期是多久？上线后的稳定性如何？有没有需要紧急回滚的情况？

### 决策类高频问题
- 为什么选这个技术栈/数据库/中间件？当时评估过哪些替代方案？
- 这个架构在什么量级下会遇到瓶颈？你当时为什么还是选了它？
- 面对技术债和新需求同时压来，你如何平衡？你推动过技术债偿还吗？
- 遇到线上紧急故障，你的处理流程是什么？你主导过哪次故障恢复？

### 方法论类高频问题
- 你如何做 code review？你会拒绝合并什么样的代码？有没有产生分歧的案例？
- 你如何评估一个重构是否值得做？什么情况下你会选择不重构？
- 你如何保证跨团队协作中接口文档和实现的一致性？
